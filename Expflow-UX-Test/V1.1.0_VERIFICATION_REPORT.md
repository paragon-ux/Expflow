## E. Confirmed v1.1.1 blockers (detail)

### V10 — API method validation

The server routes `/api/*` through `handleApi()` before checking the HTTP method. `GET /api/init` performed a real initialization. No CORS headers are returned. Localhost binding is the only guard.

Curl proves the unsafe HTTP behavior. A full browser exploitation path is not demonstrated.

**Required fixes:**
- Require `POST` for every current JSON-body API route
- Reject blank and whitespace-only roots at the GUI/API boundary
- Validate `Origin` and `Host`
- Add a per-launch request token
- Test missing-token, wrong-origin, wrong-host, GET, PUT, and OPTIONS requests
- Make initialization display and confirm the resolved absolute path

### V11 — Sync preview binding

The GUI sends only the preview's `previous_head` when committing. The runtime validates that head but then scans the working tree again, producing a fresh candidate. Committed files can differ from what the user reviewed.

**Required fixes:**
- Send `expectedCandidateDigest` alongside `expectedHead`
- Recompute the candidate under the operation lock
- Refuse commit when the digest differs from the previewed plan

### V12 — Blank root

An absent or blank root resolves to `process.cwd()`. `GET /api/init` with no body initialized the server's working directory.

**Required fixes:**
- Reject empty and whitespace-only roots at the API boundary
- Display and confirm the resolved absolute path before any mutation

### V15 — Restore preview binding

The Restore button calls `/api/restore` directly with only a reference string and `overwrite` flag. No preview token is required. No preview plan is compared before execution. The user can execute a restore they never previewed.

**Evidence:** `apps/gui/src/app.js` line 290: Restore click handler sends `{root, overwrite, reference}` directly to `/api/restore` with no preview dependency.

**Required fixes:**
- Require a current preview token before restore execution
- Recompute the restore plan and compare against what the user previewed
- Refuse execution when path effects or conflict status changed since preview
