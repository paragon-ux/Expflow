# Expflow Docs

**Status:** mutable project documentation plus immutable architecture inputs.

`docs/architecture/` is the immutable architecture-source package. Do not edit files in that directory unless the source manifest and architecture package are intentionally regenerated by an authorized architecture pass.

The rest of `docs/` is mutable implementation evidence and local orientation:

- `CURRENT_STATUS_MATRIX.md` is a live operational status file and is intentionally not treated as a stable validated document.
- `orientation/` contains mutable System 1 and System 2 pass-start controls.
- `completion_reports/` records phase and gate evidence with actual command results.
- `release_notes/` contains standalone text prepared for GitHub releases.
- `reviews/` is reserved for curated review evidence that should remain versioned.
- `REPOSITORY_DIRECTORY_STRUCTURE.md` defines the tracked, untracked, generated, package, and release-facing directory boundaries for the current release revision.
- The model documents such as `STORAGE_AND_RECOVERY.md`, `MATERIAL_RECORD_FORMAT.md`, and `AUTHORITY_AND_SEMANTIC_MODEL.md` describe the current mutable implementation stance.

Workflow phase and gate sequencing are controlled by `docs/architecture/EXPFLOW_WORKFLOW_CURRENT.md`.
