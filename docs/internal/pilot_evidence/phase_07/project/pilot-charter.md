# Pilot Charter

## Project

Phase 7 pilot evidence packet for Expflow Build Week.

## Participant Roles

- Operator: Codex, acting as the repository execution agent.
- Reviewer: later independent precision reviewer.
- Subject project: repository-owned Phase 7 evidence workspace.

## Workflow

1. Initialize Expflow in the evidence workspace.
2. Record baseline project notes.
3. Revise the task packet with observed workflow outcomes.
4. Preview pending changes.
5. Commit the material revision.
6. Verify history and restore-preview behavior.
7. Exercise conflict refusal before restore.
8. Export or otherwise record portability evidence.

## Consent and Privacy

The pilot uses only repository-owned documentation written for this phase. No human subject, customer source material, secret, credential, or personal data is collected.

## Success Criteria

- At least one real workflow task is completed or honestly abandoned with evidence.
- Command outcomes, timings, failures, retries, and decisions are recorded.
- Safety behavior is exercised without silently overwriting drift.
- Metrics are reproducible from preserved observations.
- Claims distinguish internal verification from pilot evidence.

## Stop Rules

- Stop if a command would persist a machine-absolute path.
- Stop if private data or a secret would be captured.
- Stop if restore would overwrite unrecorded work without explicit refusal evidence.
- Stop if evidence cannot support the intended claim.
